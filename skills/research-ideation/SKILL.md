# Research Ideation

From observation to research question to contribution.

## When to Use

- User says "I noticed X, is this a research question?"
- User has experimental results that suggest something interesting
- User wants to brainstorm research directions
- User needs to frame a contribution for a paper

## Status: 🚧 Draft

This skill is being developed based on ongoing research practice. Core workflow:

1. **Observation** → What did you see? (empirical evidence)
2. **Generalization** → Does this extend beyond the specific case?
3. **Literature check** → Has anyone studied this? (use related-work-survey skill)
4. **Gap identification** → What's the novel angle?
5. **Contribution framing** → What exactly are we claiming?
6. **Feasibility check** → Can we actually do this? (data, compute, timeline)
7. **Story test** → Can you explain the contribution in one sentence to a non-expert?

## Contribution Types

| Type | Example | Strength |
|------|---------|----------|
| **New problem** | "We identify a previously unstudied phenomenon" | High if well-motivated |
| **New method** | "We propose a technique that solves X better" | High if improvements are significant |
| **New benchmark** | "We provide data/evaluation for under-studied area" | Medium-high if fills real gap |
| **New analysis** | "We systematically study X and find Y" | Medium if insights are actionable |
| **New framework** | "We unify disparate approaches under one lens" | High if it changes how people think |

## Notes

- A research question is good if the answer isn't obvious AND matters to someone
- Novelty alone isn't enough — the work must be useful
- If the solution is "just prompt engineer it", it's probably not a paper
